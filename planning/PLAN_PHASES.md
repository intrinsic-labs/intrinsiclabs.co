## Development Plan Phases

We will use a rapid single-dev project plan with a load of help from Claude, my broseph from the future.

1. **Planning Phase**:
    - Define project scope
    - Create detailed project plan (see SITE_PLAN.md)
    - Configure Supabase and get Claude's help generating the site content

2. **Design Phase**:
    - Pull together reference images
    - Wireframe the site at a high level
    - Make notes on things you really want to see in the site design

3. **Frontend Phase**:
    - Build a detailed prompt for Claude to get the codebase started
    - Begin building the site, testing regularly locally
    - Build all pages defined in this document
    - Adjust Supabase schema as needed
    - Deploy the frontend site, pulling in the content from Supabase

4. **Backend Phase**:
    - Build the backend site, basing the UI on Airtable Interfaces, Github site design, and any other references I have
    - Connect it all to Supabase to manage the content
    - Build and deploy the backend site

5. **Testing and Optimization Phase**:
    - Test the site
    - Optimize the site for performance
    - Add site analytics
    - Add social media integration
    - Add SEO optimization
    - Launch the site publically

6. **Maintenance Phase**:
    - Monitor the site
    - Fix stuff that breaks
    - Listen to user feedback
    - Update the site as needed
